Full Bug Audit & Fix — All Apps
What & Why
A complete cross-artifact code audit and repair pass covering the Admin Panel, Rider App, Vendor App, Customer App (ajkmart), and API Server. The goal is to eliminate known runtime crashes, silent failures, missing error handling, unvalidated backend inputs, and broken API route mismatches — resulting in a stable, production-ready platform.

Done looks like
Admin Panel loads all pages without 404 or 500 errors from the backend
Flash Deals CRUD in admin panel works end-to-end (create, edit, delete, list)
All admin buttons that fire API calls show proper error toasts on failure — no silent failures
OTP Control "Restore Now" button handles network failures gracefully
Auth Methods page "Rotate Secret" button hits the correct backend URL
Rider login flow properly cancels all in-flight requests on unmount
Customer app Offers and Weather screens show user-visible error states instead of silent crashes
Customer app never throws unhandled promise rejections from bare fetch calls
API server routes for admin/loyalty, admin/business-rules, admin/content, and variants validate all req.body fields with Zod schemas before touching the database
product!.price non-null assertion in admin/content.ts is replaced with a proper existence check
A Puppeteer smoke-test script runs against the admin panel, clicks every sidebar nav link, and reports which routes return 4xx/5xx responses
Out of scope
Full end-to-end Cypress test suite
New features or UI redesigns
Database schema migrations beyond what is needed to fix crashes
Mobile (native iOS/Android) build fixes — web/PWA only
Steps
Fix Flash Deals admin routes (API Server) — Add GET, POST, PATCH /:id, and DELETE /:id handlers for flash deals under the /api/admin router in admin/content.ts (or a new admin/flash-deals.ts), mirroring the logic already in products.ts but gated behind admin auth middleware.

Fix auth-methods "Rotate Secret" double-prefix bug (Admin) — In artifacts/admin/src/pages/auth-methods.tsx, correct the adminFetch path from /admin/system/rotate-secret to /system/rotate-secret (the adminFetch wrapper already prepends /api/admin).

Add .catch() to all bare fire-and-forget API calls (Admin) — In otp-control.tsx line 518 (api("DELETE", "/otp/disable").then(...)), wrap in try/catch or chain .catch() to show a destructive toast on failure. Apply the same fix to cancelBypass() (line 416) and any other uncaught call found in accessibility.tsx.

Add Zod validation to unguarded admin API routes (API Server) — In admin/loyalty.ts (POST/PATCH loyalty campaigns), admin/business-rules.ts (POST), admin/content.ts (POST products), and variants.ts (PATCH), add z.object({...}).parse(req.body) guards with structured 400 error responses. Replace the product!.price non-null assertion in admin/content.ts with a proper null check + 500 response.

Fix potential undefined role assignment in auth route (API Server) — In auth/index.ts around the effectiveRole assignment, add an explicit fallback and ensure req.body.role is validated before use, so an undefined role cannot be silently stored.

Fix customer app offers.tsx and weather.tsx API calls (ajkmart) — In offers.tsx, add if (!resp.ok) throw new Error(...) before calling r.json() in all three queryFn implementations. In weather.tsx, wrap geocodeCity and handleSearchCity in try/catch and surface errors to the user via existing toast/error state.

Fix unhandled audio play rejection in customer chat (ajkmart) — In app/chat/[id].tsx around line 124, add .catch(() => {}) to audio.play() since browsers may block autoplay and throw a rejected promise.

Fix rider login abort signal not passed to sendOtp (Rider App) — In artifacts/rider-app/src/pages/Login.tsx, pass the AbortController signal to the api.sendOtp() call that follows checkIdentifier so that unmounting the component during OTP dispatch also cancels that request.

Write Puppeteer admin smoke-test script — Create scripts/admin-smoke-test.js (plain Node.js + Puppeteer) that: logs into the admin panel with seed credentials, iterates all sidebar nav links collected from the DOM, clicks each one, waits 1 second, and records the page title + any network responses with status >= 400. Outputs a markdown report to scripts/admin-smoke-report.md. Include instructions in the script's header comments on how to run it.

Verify and document all fixes — Run pnpm typecheck across the monorepo and confirm zero new type errors. Review the Puppeteer smoke report and fix any remaining 404/500 routes discovered during the run.

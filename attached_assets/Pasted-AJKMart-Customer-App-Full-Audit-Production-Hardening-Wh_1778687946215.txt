AJKMart Customer App — Full Audit & Production Hardening
What & Why
A comprehensive end-to-end audit of the artifacts/ajkmart Expo/React Native customer super-app. The goal is to eliminate every identified bug, inconsistency, and incomplete pattern — and make the full module production-ready. No changes to the API server, admin panel, rider app, or vendor app.

Done looks like
Every screen renders correctly through its full lifecycle: loading skeleton → data → empty state (when no data) → error state (with retry button) → success
All screens are wrapped in error boundaries so a crash in one screen never crashes the whole app
API_BASE is always imported from @/utils/api — no screen redefines it locally with a raw process.env expression
TypeScript compiles with zero errors on the ajkmart workspace
No screen crashes silently; all catch blocks surface an error to the user via toast or inline error state
School transport routes (/school) and school booking (/school/book) are protected by withServiceGuard and withErrorBoundary identical to the van/ride/parcel pattern
The duplicated context/constants/ directory is removed and all files that imported from it now import from constants/
Socket URLs (chat, van tracking, order tracking) are derived from a shared, reliable constant — not fragile string replacement
Wallet, profile, and parcel screens no longer silently swallow fetch errors with empty-object fallbacks
Pull-to-refresh works on every screen that loads remote data
All 130+ : any catch blocks and critical path typings are replaced with typed alternatives (unknown + type guard, or a named AppError type)
Out of scope
API server, admin panel, rider app, and vendor app — do not touch
Native EAS/OTA build config
New features not already in the codebase
Backend schema changes
Steps
Phase 1 — Infrastructure fixes (no-UI changes)
Centralise API_BASE — Delete all local const API_BASE = \https://${process.env.EXPO_PUBLIC_DOMAIN...}`/api`redefinitions in:app/_layout.tsx,app/auth/wrong-app.tsx,app/offers.tsx,app/wishlist.tsx,app/van/_Screen.tsx,app/van/bookings.tsx,app/van/seats.tsx,app/parcel/_Screen.tsx(two inline re-declarations inside async functions),components/home/ActiveTracker.tsx,components/home/StatsBar.tsx. Replace each withimport { API_BASE } from "@/utils/api". Verifyutils/api.tshandles the empty-domain case safely (returns""only when domain is missing, and never produces"https://undefined/api"`).

Derive socket URL from a shared constant — Add export const SOCKET_BASE to utils/api.ts (strips /api suffix from API_BASE safely). Replace the fragile .replace("/api", "") calls in app/chat/[id].tsx, app/chat/support.tsx, context/AuthContext.tsx, and app/van/tracking.tsx with import { SOCKET_BASE } from "@/utils/api".

Remove duplicated constants directory — Delete context/constants/ (colors.ts, rideTokens.ts, serviceRegistry.ts, typography.ts — all duplicates of constants/). Search the entire app for any imports from @/context/constants/... or relative ../context/constants/... and repoint them to @/constants/....

Replace : any with typed alternatives — Create utils/errorUtils.ts with a getErrorMessage(e: unknown): string helper (already partially done in useApiCall.ts — extract and export it). Replace all catch (e: any) { ... e.message ... } patterns across all screens and hooks with catch (e: unknown) { getErrorMessage(e) }. Fix the critical non-catch usages: handleLoginResult = async (res: any) → typed with API response type; let checkData: any → type-narrowed with unknown.

Phase 3 — Screen-level loading / empty / error states
app/my-reviews.tsx — Replace the bespoke useData hook (manual useState + useEffect with setLoading/setError) with useApiCall from @/hooks/useApiCall. Add a SmartRefresh pull-to-refresh wrapper. Show <LoadingState /> during fetch, <EmptyState /> when the reviews array is empty, and <ErrorState onRetry={...} /> on API failure.

app/recently-viewed.tsx — Currently loads from AsyncStorage but then tries to enrich items with product data. Add proper loading skeleton (<SkeletonBlock />), an <EmptyState /> for when there are no recently viewed products, and a <SmartRefresh> pull-to-refresh. Handle the case where AsyncStorage read throws (currently unguarded).

app/school/index.tsx — Replace raw ActivityIndicator loading with <SkeletonBlock /> rows. Add <EmptyState /> for no routes. Surface fetch errors inline with <ErrorState onRetry={...} /> instead of only Alert.alert.

app/school/book.tsx — Wrap the booking submit in a full try/catch using useApiCall. Show a toast on success, a typed error message on failure. Disable the submit button while submitting === true and show an ActivityIndicator inside it.

app/restaurants/index.tsx — Add loading skeleton, empty state for no restaurants, and error state with retry. Add SmartRefresh pull-to-refresh.

app/weather.tsx — Wrap the weather fetch in useApiCall to gain automatic retry and toast-on-error. Show a graceful <ErrorState> card (not just silent failure) if the weather API returns no data.

app/van/tracking.tsx — The WebView-based map has no fallback if WebView fails to load or the socket connection drops. Add a onError handler on the <WebView> that shows an <ErrorState> overlay. Add a socket reconnect notice if the van location stops updating for > 60 seconds.

Phase 4 — Silent error fixes
app/(tabs)/profile.tsx silent swallows — Lines 483–484: pharmacy and parcel fetch failures silently return {} via .catch(() => ({} as ...)). Replace with explicit catch blocks that set per-section error state and show a dismissible inline retry banner for each section, rather than swallowing the error.

app/(tabs)/wallet.tsx duplicate deposit-methods fetch — deposit-methods is fetched in two separate useEffect calls (lines 685 and 1009/1192). Consolidate into a single useQuery so data is cached and not double-fetched. Fix the withdrawal-methods fetch which has no error handler — add a .catch that shows a toast.

app/parcel/_Screen.tsx inline API_BASE — Two async functions re-declare API_BASE as a local const (lines 201, 243). Remove these and use the module-level import added in Step 1.

app/orders/[id].tsx socket fallback — The dynamic import("socket.io-client") inside useEffect has no rejection handler. Add .catch((err) => log.warn("socket load failed", err)) and a visible "Live tracking unavailable" notice when the socket module fails to load.

app/chat/[id].tsx token refresh on 401 — Outgoing chat API calls use a raw fetch with a Bearer token. If the token expires mid-session the request silently fails with HTTP 401. Add a 401 check after each fetch that calls refreshToken() from AuthContext and retries once before surfacing an error.

Phase 5 — TypeScript cleanliness & build verification
Fix all remaining TypeScript errors — Run npx tsc --noEmit inside artifacts/ajkmart. Fix any errors introduced by the earlier refactors (likely: import paths after context/constants deletion, removed any casts, new strict types on catch blocks). The build must exit cleanly with 0 errors.

ESLint pass — Run pnpm --filter @workspace/ajkmart lint (using the existing eslint.config.js). Fix all reported errors. Warnings may remain if they are pre-existing and unrelated to this task.

Full Codebase Bug Fix — All 5 Apps
What & Why
A comprehensive scan of all five apps (Admin Panel, Rider App, Vendor App, Customer App/ajkmart, API Server) revealed critical bugs across routes, components, contexts, and middleware. These bugs include race conditions, broken cancellation flows, undefined variables, incorrect socket usage, financial precision errors, auth inconsistencies, and more. This task fixes all of them to bring the entire platform to a fully working, bug-free state.

Done looks like
Admin panel order cancellation works correctly without duplicate dialogs or wrong mutations
Missing status colors (ready, picked_up) show correctly in the Admin UI
useCategories in admin uses the auth-wrapped fetcher (not raw fetch)
Rider app Home.tsx has no undefined variable crashes (totalRequests fixed)
Rider Chat.tsx uses .off() instead of removeAllListeners to avoid breaking shared socket
Vendor app geolocation effect does not fire redundant API calls on every update
Vendor image/audio/video uploads all use consistent payload format
Customer cart validation race condition resolved — save and validate do not overwrite each other
Cart ack timer does not prematurely clear cart on partial status updates
AuthContext JWT decode handles all base64 padding edge cases
API server rider routes all sit behind the riderAuth middleware correctly
Financial calculations across rider earnings use integer cents or big.js to avoid floating-point drift
OSRM failure falls back to Haversine distance gracefully
IDOR guard covers orderId, rideId, not just userId
No double toasts on API errors in admin
Token refresh loop handles slow-network 401 scenario correctly
AdaptiveImage shows a placeholder on 404/500 instead of blank boxes
Out of scope
New features or UI redesigns
Database schema migrations
Third-party service integrations (Stripe, Firebase setup, etc.)
Performance optimizations unrelated to bugs
Steps
API Server — Auth & Route Ordering: Audit all rider and vendor route files to ensure every sensitive route sits after its auth middleware. Standardize request user property to req.riderUser, req.vendorUser, req.customerId consistently across all middleware.

API Server — Financial Precision: Replace floating-point arithmetic in rider earnings/commission calculations with integer-cents math or big.js. Fix the hardcoded rider_keep_pct = 80 fallback to always read from the platform settings table.

API Server — OSRM & IDOR Fixes: Add a graceful Haversine fallback when OSRM returns an error or is unreachable. Extend the IDOR guard to also check :orderId, :rideId, :vendorId params in addition to :id / :userId.

Admin Panel — Order Cancellation Flow: Fix handleCancelOrder to call the correct cancellation mutation/endpoint instead of the generic updateMutation. Ensure the status dropdown and "Cancel Order" button do not conflict, and that the React Query cache is invalidated properly after cancellation.

Admin Panel — Missing Status Colors & Cache Sync: Add the missing ready and picked_up cases to getStatusColor in lib/format.ts. Fix dashboard.tsx to invalidate the admin-platform-settings query key after saving thresholds, and fix the hardcoded threshold strings "3" / "30" to be typed as numbers.

Admin Panel — Auth Bypass & Double Toasts: Replace the raw fetch in useCategories with the adminFetch wrapper. Remove the redundant .js extension in adminFetcher.tsx import. Suppress the double-toast on API errors (keep only the component-level toast, remove the global timeout toast for those routes).

Rider App — Undefined Variables & Socket Fix: Define totalRequests correctly in Home.tsx as orders.length + rides.length. Replace all socket.removeAllListeners("event") calls in Chat.tsx with socket.off("event", handlerRef) to avoid removing other components' listeners.

Rider App — Sound Interval & GPS Optimisation: Stabilize the sound interval in Home.tsx so it doesn't create overlapping intervals when multiple requests arrive. Reduce double-traffic by coordinating socket heartbeat location updates with the REST updateLocation calls.

Vendor App — Upload Payload Consistency: Unify uploadImage, uploadAudio, and uploadVideo in api.ts to all use FormData (multipart), matching what the API server's multer middleware expects.

Vendor App — Geolocation Effect & Socket Path: Fix the geolocation useEffect in Orders.tsx to only call saveVendorLocationToBackend when coordinates actually change (add a comparison before dispatching). Fix the hardcoded /api/socket.io Socket.IO path to respect the app's base path.

Customer App — Cart Race Condition & Ack Timer: Fix the CartContext validation race by buffering the save until validation completes. Fix startAckStuckTimer to not clear the cart until the order status moves to a finalized state (confirmed/failed), not just any non-pending state.

Customer App — AuthContext & Token Refresh: Fix JWT base64 decode in AuthContext to properly pad the string before atob. Add a safety buffer to scheduleProactiveRefresh so that on slow networks it retries earlier rather than allowing a 401 to occur.

Customer App — API Client Consistency: Replace all hardcoded fetch(${API_BASE}/...) calls in cart/index.tsx with the @workspace/api-client-react wrappers to get automatic token injection and circuit-breaker support.

Customer App — AdaptiveImage & Role Block: Add a onError fallback image/placeholder to AdaptiveImage.tsx for 404/500 responses. Fix AuthContext logout to reset the "role denied" state so users are not trapped on the block screen after re-login.
